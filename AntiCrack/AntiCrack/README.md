#  Anti-Crack System

##  Төслийн тухай (About the Project)
Энэхүү төсөл нь програм хангамжийг "Reverse Engineering" (Ухраан задлах инженерчлэл) болон хууль бус өөрчлөлтөөс (Cracking/Patching) хамгаалах зорилготой олон давхаргат (Multi-layered) хамгаалалтын систем юм. C/C++ хэл болон Windows API ашиглан үйлдлийн системийн доод түвшинд (low-level) ажиллах хамгаалалтын механизмуудыг хэрэгжүүлсэн.

##  Ашигласан технологиуд (Tech Stack)
* **Хэл:** C / C++
* **Орчин:** Microsoft Visual Studio 2022
* **Үйлдлийн систем:** Windows 11
* **Технологи:** Windows API, Memory Management, Cryptography (XOR/Hashing)
* **Тестлэх хэрэгслүүд (Крэкерүүдийн хэрэгсэл):** x64dbg, IDA Pro, Cheat Engine

##  Төслийн хэрэгжүүлэх үе шатууд (Roadmap & Implementation Plan)

### Phase 1: Target Application (Бай програм үүсгэх)
Хамгаалалтын системийг турших зориулалттай үндсэн програм. 
- [x] Хэрэглэгчээс нууц үг/лиценз нэхдэг систем бичих
- [x] Зөв болон буруу хариуг ялгаж ажилладаг логик оруулах

### Phase 2: Anti-Debugging Techniques (Дебаггерийн эсрэг хамгаалалт)
Крэкерүүд дебаггер (x64dbg) ашиглан програмын ажиллагааг зогсоож, алхам алхмаар шинжлэхээс сэргийлэх.
- [ ] `IsDebuggerPresent()` функц ашиглан дебаггерийг илрүүлэх
- [ ] `CheckRemoteDebuggerPresent()` ашиглан гадны процесс хянаж байгааг шалгах
- [ ] `PEB` (Process Environment Block) бүтцийг санах ойд шууд шалгах (Advanced)

### Phase 3: Code & String Obfuscation (Код болон өгөгдөл будилаантуулах)
Програм доторх чухал мэдээллийг (жишээ нь: "Secret123" гэх мэт нууц үг) статик шинжилгээгээр (Static Analysis) шууд унших боломжгүй болгох.
- [ ] Чухал тэмдэгт мөрүүдийг (Strings) XOR алгоритмаар шифрлэх
- [ ] Програм ажиллах үед (Runtime) санах ойд буцааж тайлах (Decryption)

### Phase 4: Integrity / Hash Check (Бүрэн бүтэн байдлын шалгалт)
Програмын `.exe` файлыг Hex Editor ашиглан өөрчлөх (Patch хийх) оролдлогыг таслан зогсоох.
- [ ] `.exe` файлын өөрийнх нь Hash (MD5 эсвэл SHA-256) утгыг тооцоолох
- [ ] Програм асах бүрт Hash утгыг шалгаж, 1 байт л өөрчлөгдсөн байвал өөрийгөө хаах

### Phase 5: Anti-Dumping & Anti-VM (Нэмэлт хамгаалалт)
- [ ] Програмыг санах ойноос хуулж авахын (Memory Dump) эсрэг хамгаалалт
- [ ] Virtual Machine (VMware, VirtualBox) дотор ажиллаж байгааг мэдэрч хаагдах

##  Хүлээгдэж буй үр дүн (Expected Outcomes)
Энэхүү төсөл дууссанаар энгийн C/C++ програм нь аливаа Reverse Engineering болон Malware Analysis хэрэгслүүдийн эсрэг хүчтэй эсэргүүцэл үзүүлж чадах бөгөөд Мэдээллийн Аюулгүй Байдлын (Information Security) чиглэлээр доод түвшний (Low-level) ойлголтуудыг бүрэн харуулах болно.
